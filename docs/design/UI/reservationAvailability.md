# 予約状況画面 UI設計

## 1. 画面概要

ログイン後に最初に表示される画面。  
駐車場の予約状況を時間帯ごとに確認し、空き枠の予約を行う。

駐車枠 × 時間（30分単位）のタイムライン形式で表示する。

---

# 2. 表示内容

## 2.1 予約検索エリア

画面上部に予約検索条件を表示する。

|項目|説明|
|---|---|
|開始日時|予約希望開始日時|
|終了日時|予約希望終了日時|
|検索ボタン|条件に合う予約可能枠を表示|

日時は **日付＋時刻** を指定する形式とする。  
日付をまたぐ予約にも対応する。

例

```
開始日時
[2026-03-14 09:00]

終了日時
[2026-03-14 10:30]

[検索]
```

---

# 3. 予約状況タイムライン

駐車枠ごとの予約状況をタイムライン形式で表示する。

|項目|説明|
|---|---|
|縦軸|駐車枠|
|横軸|時間（30分単位）|

時間表示は **その日の0:00〜24:00** を基本とする。  
横スクロール可能とし、日付をまたぐ予約も表示可能とする。

---

## 表示例

```
時間 →

        0:00 0:30 1:00 ... 9:00 9:30 10:00 10:30 ...

駐車枠1   □   □   □   □   □   ■   ■   □
駐車枠2   □   □   ■   ■   □   □   □   □
駐車枠3   □   □   □   □   □   □   □   ■
```

---

# 4. スロット状態表示

|状態|色|説明|
|---|---|---|
|空き|薄青|予約可能|
|他人予約|グレー|予約不可|
|自分予約|緑|自分の予約|

他人の予約スロットはクリック不可とする。

---

# 5. 予約検索時の表示

検索条件に **開始日時・終了日時** を入力し検索を行った場合、  
以下の条件を満たす駐車枠のみ表示する。

条件

```
希望時間の全てのスロットが空きであること
```

例

希望時間

```
09:00〜10:30
```

必要スロット

```
09:00
09:30
10:00
```

すべて空きの駐車枠のみ表示する。

---

# 6. 予約可能枠表示

条件を満たす駐車枠には、希望時間帯を **1つの予約ボタンとして表示**する。

例

```
駐車枠3

[ 09:00〜10:30 予約する ]
```

このボタンをクリックすると **予約作成画面へ遷移する。**

---

# 7. スクロール仕様

時間軸は横スクロール可能とする。

理由

- 30分単位で48スロット存在する
- 日付をまたぐ予約を表示可能にするため

---

# 8. 操作フロー

|操作|動作|
|---|---|
|空き枠クリック|予約作成画面へ遷移|
|自分の予約クリック|予約詳細表示|
|他人の予約クリック|操作不可|
|検索条件入力|予約可能枠を再表示|

---

# 9. 初期表示

ログイン後は

```
本日の予約状況
```

を表示する。

検索条件は未入力状態とする。

---

# 10. UI補足

- 時間表示は30分単位
- 横スクロール対応
- 色による状態判別
- 日付をまたぐ予約対応
